Paradigma

Un paradigma de programación es un estilo o enfoque para resolver problemas con código. Java es principalmente un lenguaje orientado a objetos (OOP): organiza el código en clases y objetos, usando conceptos como herencia, encapsulamiento y polimorfismo. Python soporta múltiples paradigmas: OOP, imperativo y funcional.

Clase

Una clase es la plantilla para crear objetos. Define atributos (estado) y métodos (comportamiento).

// Java
public class Persona {
  private String nombre;
  private int edad;

  public Persona(String nombre, int edad){
    this.nombre = nombre;
    this.edad = edad;
  }

  public void saludar(){
    System.out.println("Hola, soy " + nombre);
  }
}

Método - Atributos

Atributos (fields) almacenan datos; métodos realizan acciones. Los métodos tienen firma (tipo de retorno, nombre y parámetros).

// atributo y método
private int contador;      // atributo
public void incrementar(){ // método
  contador++;
}

Visibilidad: public / private / protected

public: accesible desde cualquier clase. private: sólo dentro de la misma clase. protected: dentro del mismo paquete y subclases.

public class A { public int x; protected int y; private int z; }

Encapsulamiento (getters y setters)

Consiste en ocultar los datos (atributos privados) y exponer acceso controlado mediante getters/setters.

private String nombre;
public String getNombre(){ return nombre; }
public void setNombre(String n){ this.nombre = n; }

Constructor - Sobrecarga de constructores

El constructor inicializa objetos. Puedes tener varios constructores con distintas firmas (sobrecarga).

public Persona(){ /* defecto */ }
public Persona(String n){ this.nombre = n; }
public Persona(String n, int e){ this.nombre = n; this.edad = e; }

Herencia

Permite que una clase subclase extienda a otra (superclase), heredando atributos y métodos.

public class Animal { void comer(){} }
public class Perro extends Animal { void ladrar(){} }

Agregación

Es una relación "tiene-un". Un objeto contiene referencias a otros objetos. Si la vida del objeto contenido no depende del contenedor, es agregación (si depende, sería composición).

public class Motor {}
public class Auto { private Motor motor; public Auto(Motor m){ this.motor = m; } }

Métodos abstractos - Clases

Una clase abstracta puede tener métodos sin implementación (abstractos). No se pueden instanciar directamente.

public abstract class Figura {
  abstract double area();
}
public class Circulo extends Figura {
  private double r;
  public double area(){ return Math.PI * r * r; }
}

Clase / Métodos / Atributos final

final evita la modificación: una clase final no puede extenderse; un método final no puede sobrescribirse; un atributo final es constante (inmutable después de asignación).

public final class Util { public static final double PI = 3.14159; }

Interfaces

Una interfaz declara métodos (sin implementación, aunque desde Java 8 puede tener default/static). Una clase implementa interfaces y se compromete a definir sus métodos.

public interface Volador { void volar(); }
public class Pajaro implements Volador { public void volar(){ /* ... */ } }

Excepciones

Mecanismo para manejar errores en tiempo de ejecución. try / catch / finally. Java distingue checked exceptions (requieren ser declaradas/atrapadas) y unchecked (runtime).

try {
  int r = 10 / divisor;
} catch (ArithmeticException e) {
  System.out.println("Error: división por cero");
} finally {
  // se ejecuta siempre
}

Hilos (Threads)

Permiten ejecutar tareas concurrentes. Se puede extender Thread o implementar Runnable. Controlar sincronización es clave para evitar condiciones de carrera.

Thread t = new Thread(() -> {
  for(int i=0;i<5;i++) System.out.println(i);
});
t.start();

Listas

Colección ordenada. En la API de colecciones de Java se usan interfaces como List y clases como ArrayList o LinkedList.

List<String> nombres = new ArrayList<>();
nombres.add("Ana");
for(String n : nombres) System.out.println(n);

Enums

Sirven para definir conjuntos de constantes con nombre y se pueden enriquecer con campos y métodos.

public enum Dia { LUNES, MARTES, MIERCOLES }

JavaFX

Framework para construir interfaces gráficas modernas en Java (reemplazante del antiguo Swing en muchos casos). Usa Stage, Scene y nodos como Button, Label.

// Estructura mínima
public class App extends Application {
  public void start(Stage stage){
    Button b = new Button("Hola");
    stage.setScene(new Scene(new StackPane(b), 300, 200));
    stage.show();
  }
}

Genéricos

Permiten tipos parametrizados (seguridad en tiempo de compilación). Ej: List<String> evita casts y errores de tipo.

public class Caja<T> { private T valor; public void set(T v){ valor = v; } public T get(){ return valor; } }

Serialización y manejo de archivos

Serializar convierte objetos a bytes para almacenarlos o transmitirlos. En Java: Serializable y ObjectOutputStream/ObjectInputStream. Para archivos, usar Files o streams.

try(ObjectOutputStream oos = new ObjectOutputStream(new FileOutputStream("obj.dat"))){
  oos.writeObject(miObjeto);
}

Python: Estructuras de control — decisiones y ciclos

Condicionales con if/elif/else. Bucles con for y while. Python usa indentación para los bloques.

# if
x = 10
if x > 5:
  print('mayor')

# for
for i in range(5):
  print(i)

# while
while x > 0:
  x -= 1

Python: Diccionarios, Tuplas, Listas

Listas son colecciones ordenadas y mutables. Tuplas son como listas pero inmutables. Diccionarios son mapas clave→valor.

# lista
lista = [1,2,3]
# tupla
tupla = (1,2)
# diccionario
dic = {'nombre':'Ana', 'edad':30}
print(dic['nombre'])
